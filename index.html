01. IoT 기반 전동 킥보드 다인 탑승 감지 시스템
+

[Creative Innovation Project]

1. 프로젝트 개요 (Problem)

전동 킥보드 2인 이상 탑승 사고가 심각한 사회적 문제입니다. 기존의 단순 경고 방식을 넘어, 실제 위험 상황만을 정확히 감지하여 제어하는 시스템을 개발했습니다.

2. 핵심 솔루션 (Logic)

단일 센서의 오작동을 막기 위해 센서 융합(Sensor Fusion) 기술을 적용했습니다.

  • 압력 감지: 발판의 FSR 센서 4개 중 3개 이상 동시 인식
  • 거리 감지: 초음파 센서로 탑승자의 앞뒤 간격 밀집도 분석
  • 판단 알고리즘: 위 두 조건이 AND로 3회 이상 연속될 때만 위험으로 간주 (오탐 최소화)

3. 하드웨어 구성 (Hardware)

  • MCU: Arduino Uno
  • Sensors: FSR 압력센서 4EA, 초음파 센서 4EA
  • Output: Buzzer (경고 출력)

4. 성장 포인트 (Growth)

단순히 센서 값을 읽는 코딩이 아니라, '어떻게 해야 실수로 울리지 않을까?'를 고민하며 데이터 누적 판단 알고리즘을 설계하는 엔지니어링 사고력을 길렀습니다.

02. P2P 블록체인 기반 탈중앙 메신저 설계
+

[Deep Dive Challenge - 탐구 프로젝트]

1. 문제 상황 인식 (Problem Definition)

카카오톡 데이터센터 화재 사건을 보며, 중앙 집중형 메신저가 가진 단일 실패 지점(SPOF)의 위험성을 깨달았습니다.
"서버가 멈추면 우리의 대화도 멈춰야 하는가?"라는 근본적인 질문(5 Whys)을 던졌고, 중앙 서버 없이 작동하는 메신저가 필요하다는 결론을 내렸습니다.

2. 탐구 설계 (Golden Circle)

  • Why (Vision): 재난, 검열, 서버 마비 상황에서도 개인의 소통과 정보 주권은 지켜져야 합니다.
  • How (Strategy): 기업의 중앙 서버를 제거하고, 사용자 컴퓨터끼리 직접 연결하는 P2P 네트워크를 구축합니다. 또한 데이터 신뢰성을 위해 블록체인 기술을 적용합니다.
  • What (Result): Python 기반의 탈중앙화 메신저 프로토타입 구현 및 검증.

3. 기술적 구현 (Implementation)

  • P2P Socket 통신: Python Socket 라이브러리를 활용해 서버를 거치지 않고 Client to Client 직접 통신 환경을 구축했습니다.
  • 블록체인 구조: 채팅 메시지를 하나의 Block으로 정의하고, SHA-256 해시 함수로 이전 대화와 현재 대화를 체인처럼 연결했습니다. 이를 통해 대화 내용의 위·변조가 불가능함을 기술적으로 증명했습니다.

4. 결론 및 가치 (Impact)

단순히 코드를 짜는 것을 넘어, 기술이 사회적 문제(통신 단절)를 어떻게 해결할 수 있는지 고민했습니다.

"기업의 서버가 멈추어도, 사람들의 소통은 계속되어야 한다."

이 프로젝트를 통해 네트워크 구조에 대한 깊은 이해와 공학적 문제 해결 역량을 길렀습니다.

03. 컴퓨팅 사고력과 알고리즘 탐구
+

[Algorithm & Data Structure]

1. 활동 내용

효율적인 문제 해결을 위해 백준(BOJ) 및 정보 올림피아드 기출 문제를 분석하고 풀이했습니다. 단순히 정답을 맞히는 것보다 '더 빠른 시간 안에' 해결하는 최적화에 집중했습니다.

2. 학습 포인트

  • 시간 복잡도 최적화: 이중 반복문으로 작성된 O(n^2) 코드를 스택 자료구조를 활용해 O(n)으로 단축시키는 경험을 했습니다.
  • 논리적 사고: 프로세스 스케줄링 문제를 큐(Queue)를 활용해 해결하며, 컴퓨터가 데이터를 처리하는 순서를 논리적으로 설계하는 힘을 길렀습니다.
