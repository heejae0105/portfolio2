01. IoT 기반 전동 킥보드 다인 탑승 감지 시스템
▼

[Creative Innovation Project]

1. 프로젝트 개요 (Problem)

전동 킥보드 2인 이상 탑승 사고가 사회적 문제로 대두되었습니다. 단순 경고가 아닌 기술적 제어를 통해 실제 위험 상황만을 정확히 감지하는 시스템을 목표로 했습니다.

2. 핵심 솔루션 (Solution Logic)

단일 센서의 오작동 한계를 극복하기 위해 센서 융합(Sensor Fusion) 방식을 채택했습니다.

  • 조건 1 (압력): FSR 센서 4개 중 3개 이상 감지
  • 조건 2 (거리): 초음파 센서(전/후) 거리 합이 기준치 이하
  • 알고리즘: 위 두 조건이 AND로 3회 이상 연속 충족될 때만 '위험'으로 판단하여 오탐을 최소화했습니다.

3. 하드웨어 구성 (Hardware)

  • Controller: Arduino Uno
  • Sensors: FSR 압력센서, 초음파 센서
02. P2P 블록체인 기반 탈중앙 메신저 설계
▼

[Deep Dive Challenge - 탐구 프로젝트]

1. 문제 상황 인식 (Problem Definition)

중앙 집중형 메신저의 데이터센터 화재로 인한 통신 마비 사태를 보며, 단일 실패 지점(SPOF)의 위험성과 정보 주권 침해 문제를 인식했습니다.

"왜 서버가 멈추면 우리의 대화도 멈춰야 하는가?"라는 질문(5 Whys)을 통해, 중앙 서버 없이 연결되는 구조가 본질적 해결책임을 도출했습니다.

2. 탐구 설계 (Golden Circle)

  • Why (Vision): 재난이나 검열 상황에서도 개인의 소통은 중단되지 않아야 합니다.
  • How (Strategy): 중앙 서버를 제거하고 사용자 간 직접 연결(P2P)을 수행하며, 신뢰성 확보를 위해 블록체인 기술을 적용합니다.
  • What (Result): Python 기반의 탈중앙화 메신저 프로토타입 구현.

3. 기술적 구현 (Implementation)

  • P2P Network: Python Socket 프로그래밍으로 서버 없이 클라이언트 간 직접 통신 환경 구축.
  • Data Integrity: 채팅 메시지를 Block 단위로 생성하고, SHA-256 해시 함수로 이전 블록과 연결(Chaining)하여 데이터 위·변조 불가능성 증명.

4. 결론 및 가치 (Impact)

단순한 코딩을 넘어 "기업의 서버가 멈추어도 사람들의 소통은 계속되어야 한다"는 기술의 사회적 가치를 실현했습니다.
