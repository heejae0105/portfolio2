01. IoT 전동 킥보드 2인 탑승 감지
▼ 자세히 보기
제작 동기 및 문제 인식

학교 등하굣길에서 전동 킥보드 하나에 두 명이 위태롭게 매달려 가는 모습을 자주 목격했습니다. 단순한 경고문보다 기계가 스스로 인원을 감지하고 경고음(피드백)을 주는 시스템이 있다면 사고를 실질적으로 줄일 수 있을 것이라 판단했습니다.

기술적 구현 (Sensor Fusion)
Arduino Uno C++ FSR(압력) 초음파센서

단일 센서만으로는 가방을 올려두거나 주행 중 진동을 사람으로 오인하는 '오탐(False Positive)' 문제가 발생했습니다. 이를 해결하기 위해 두 종류의 센서 데이터를 결합하는 센서 융합(Sensor Fusion) 알고리즘을 설계했습니다.

  • 조건 A (압력): 발판 FSR 센서 4개 중 3개 이상 감지
  • 조건 B (밀집도): 초음파 센서 거리 합이 기준치 이하 (밀착 상태)
  • 검증: 위 두 조건이 '3회 연속' 감지될 때만 최종 경고 출력
02. DDC 창의탐구: P2P 블록체인 메신저
▼ 자세히 보기
탐구 배경: 단일 실패 지점(SPOF)

카카오톡 데이터센터 화재 사건을 보며 '중앙 집중형 서버'의 취약점을 깊이 체감했습니다. 이에 중앙 서버가 파괴되더라도 개인 간 연결이 유지되는 '서버 없는(Serverless) P2P 메신저'를 기획했습니다.

핵심 기술: Socket & Blockchain
Python Socket Programming SHA-256

[탈중앙화 통신] 거대 서버를 거치지 않고 Python Socket 라이브러리를 활용해 IP 주소만으로 직접 데이터를 주고받는 환경을 구축했습니다.

[데이터 무결성] 중앙 관리자가 없으면 대화 내용 위변조 위험이 생깁니다. 이를 막기 위해 이전 메시지의 해시값(SHA-256)을 현재 메시지에 포함시키는 체인(Chain) 구조로 설계하여 보안성을 확보했습니다.
